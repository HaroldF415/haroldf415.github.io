Parallax Scrolling

Still following some tutorials from Travis Neilson at DevTips and the one that caught my eye was the Parallax Scrolling series. At first hearing this term I was confused as to what it was. To be honest I thought it had something to to with Parallax the fictional comic book supervillain in the DC Comics Universe. Anyways it is far from that. Apparently the term "Parallax" was given to the visual effect of 2D side scrolling videogames that moved different background _img at different movement speeds to create and illusion of depth. This was achieved by making the background of said video game move at a slower speed than the foreground to give it that effect of it being further away. This same idea is used and applied to site design. In this case web designer move the background and foreground of a site at different speeds than the rest of the page. There are many reasons why web designers implement Parallax Scrolling to their sites. For me I'm going to implement it to my site for practice. I've added the code to my GitHub here. To start I looked for pictures that had a good background that I could blur out in Photoshop. After I looked for a clean image that I could extract and put in the foreground of the background. What is Node.js?

Simply put Node.js is JavaScript that has access to sockets, databases, files, etc. The same way other languages like PHP, Python, or Ruby have always been able to do. To actually understand this better ( myself included ) I go back to JavaScript's Execution Context.

What is Execution Context?

The environment in which JavaScript code is run is very important, and is evaluated as one of the following types of code:

  • Global - Default environment where code is executed for the first time.
  • Function - Whenever the flow of execution enters a function body.
  • Eval - Text to be executed instead of the internal eval function.

Global context is highlighted in purple. Function context is highlighted in green, blue and orange. Global context can be accessed from any other context in your code. There can be any number of function contexts, meaning each one of these function calls can create a new context. This creates a scope where anything declared within said function cannot be directly accessed from the outside.

The example above has a function that can access a variable declared outside of its scope, but an outside context can not access the variables / declared inside.

  • If we are inside the function called Person() we can access the global variable printHello.
  • If we are inside the purple area, we can not access variables like firstName or lastName.

The Execution Context is where code is executed from. There is the Global Execution Context, and there is Function Execution Context. A JavaScript program always starts with the Global Execution Context, and when a function is called, it enters that function's Execution Context. This context is added to the "Call Stack", which can be visualized as layers. So now you have two layers, the Global Execution Context and one Function Context.

There are many things to say about the Execution Context but that will be all for now. Next getting back to installing NPM.
