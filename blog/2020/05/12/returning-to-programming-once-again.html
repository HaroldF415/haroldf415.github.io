Returning to Programming Once Again.

Posted on: May 12th, 2020

It's been a couple of months since I last updated this website. Hate to admit it but time management has not been my forte. Every time I open up WebStorm to start programming again I look at it as a rubik's cube. A lot has happened since I last posted. The world has changed a lot. My life has changed a lot. My son now walks and runs everywhere. I can barely keep up. But it seems that I keep coming back to programming as a sort of release. It feels good to write code, tinker with small attributes of a website and make it....work.

Writing a blog about my experience as a programmer lets me keep track of where I am. I still love to code. From time to time I peek at snippets of code and try to figure out what they do. Most of the time I'm lost but it always intrigues me. This time I hope to change things up and laser focus on what exactly it is that I want to do with programming.

First thing I must do is to get used to using git again. Must remember that every new file that is added to my project must first be added to the staging area by git add . or git add -A. This will take all the new and untracked files created and add them to the staging area. After this use the command git commit -a -m "random msg" this will commit all tracked files and gets them ready for git push.
