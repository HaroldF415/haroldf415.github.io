Website structure, refactoring code.

Posted on: February15, 2016

Every time I go back to check up on my website I see ways it could become more efficient. So I decided to restructure the way my files were setup.

I started with the stylesheets that were a horror to look at....well at least for me it was.

When I learned how to use Sass I was amazed at how you could have separate files styling one html page. Of course all those Sass files would later be imported and then compiled to one giant css file.

I found that separating the style of each section of my page was not productive but more time wasting. For example if i had to look for an error I would have to dig through folder after folder.

So what I decided to do next was that each html file would have its own stylesheet, only one. Within that one stylesheet I would then separate the code with comments.

This would not only cut the time I spent looking for my error in half but also decrease the amount of files needed in my main project folder.
