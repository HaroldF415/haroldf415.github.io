Third Day Blog Entry.

Posted on: January 30, 2019

This is the third entry for my blog after a long hiatus. Programming once again feels good. Muscle memory is kicking in and writing simple mark-up for the blog comes naturally by now. Checking over the code I have a hang of it now. Every time I wish to add another blog I must copy and paste the code from the previous post. Must find a simpler way to do this. Even though there are much simpler tools out there to write a simple blog, I feel that forcing myself to look and write code is beneficial. Two things must be added to the site when writing another blog post. First go over to blog.html add a snipped of code called the blog_entries_preview this as mentioned in the previous post will hold the information of when the post was made, a small preview image of what the post is about and the title of the post. Second thing to do is to go over the code that I will copy and paste for each blog entry. Write small comments that will let me know to change the code because some of it will have the date from the previous post. Not only will it have the date of the previous post but it will also link to other files that are only specific to that post. Like links and time stamps.

Writing down small comments on the side of code that needs to be changed is taking a little bit more time than I expected it to be. There is a constant need to get everything precisely like I want it or else I won't continue with the next step. This I believe to be my greatest hurdle to overcome. I've spent hours on something that wasn't to my liking and could not move past it unless I fixed it. For now I will end the post and continue it later. Every time I add a blog post I create a new folder that contains the HTML file of the blog entry. When submitting it to Github I must remember that the following line of code does not work for some reason.

git commit -a -m "random description about the files being committed"

The reason why I use the code above is because it came up when I was reading the book Pro Git by Scott Chacon and Ben Straub. Decided to give the book another look to find out what exactly this line of code did. Could not get it out of my head as to why after adding another folder (directory) and another html the line of code did not work.

" If you want to skip the staging area, Git provides a simple shortcut. Adding the -a option to the git commit command makes Git automatically stage every file that is already tracked before doing the commit, letting you skip the git add part."

– Scott Chacon and Ben Straub, Pro Git

When I started with this project of my portfolio website I looked into may different forms on how to do it. As of right now I'm using Webstorm 2018.2 and CodeKit Version 3.7.4 -- Build 28905. It can get pretty confusing trying to manage all of them at the same time just to push out content to a simple blog website. Anywho getting back to the task at hand which was why the code above wasn't working.

5:21pm - my time programming is coming to an end so I will do this quickly.

git status

git commit -a -m "adding blog entry 01_30_19"

Random Thoughts:

  • Also stop going back and forth with blog post and blog entry. Pick one
  • Look into markup language again. I feel that I'm using the p tag constantly to get some space between what I'm saying and code that I want to show.

5:35pm - I will now end the post and add it to the remote repository with the correct git commands. Will continue with adding the comments to the copy/paste idea later tonight.
