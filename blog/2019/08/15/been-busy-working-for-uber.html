Been Busy Working For Uber.

Posted on: August 15th, 2019

Over the past two days I have not submitted nothing to my website because of my main job that is working with Uber/Lyft. This job takes the majority of the time during the day because of the amount of hours I must spend on the streets to make a decent payday. I have a family to provide for and putting money in the bank seems to be the priority for now. I've figured out how to create templates to fit my own needs in WebStorm. Especially when it comes to my blog entries. Instead of copying and pasting I just created a template that held all the code I need for an entry like this one. The only thing I must do is go inside and change the dates of when this was posted. There must be an easier way to accomplish this but for now I will leave it as is. Repeating these steps time and time again will make it easier for me to remember what I must do.

Adding new folders or files to the project must be added to my local repository before committing them to my remote repository. So there was a mix up with me committing files to my remote directory. I was trying to use a shortcut 

                
                    $ git commit -a -m "some weird description"

Git provides this shortcut which adds the -a option to the git commit command which then makes Git automatically stage every file that is already tracked before doing the commit. I believed this shortcut was to add newly created files from my local repository into my remote repository. But now that I've read up on this using the Pro Git Book on pg. 36 I understand much better what that shortcut does.

Looking at various ways to improve my personal website. Firstly by looking at ways to present my code like the one above. Maybe I'm just being a perfectionist but I can't seem to find a guide on how to exactly do all this. Right now I will begin with writing things down about Git. In other blog entries I've mentioned that I'm reading the Pro Git Book to get a better understanding on how this VCS works.
