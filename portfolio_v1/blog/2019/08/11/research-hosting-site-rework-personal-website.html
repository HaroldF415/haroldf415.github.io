Research Hosting Site & Rework Personal Website.

Posted on: August 11th, 2019

Getting into the habit of writing personal blogs again to keep track of my progress. I'm doing this the tedious way by that I mean that I have to write code consistently. updating every single blog entry I make individually. This is not efficient I know, but it helps me get used to the code and figure out what it does. So that in the next blog entry I write is much easier to get right. This time I am also researching which web hosting company is more beneficial for me. GithubPages has served me well, but in the real world I wont be using GithubPages. For this reason I must get used to working with other forms of hosting a website. Learn the ins and outs of web development. Once I have that down I will rework my website because its been long overdue and it quite honestly feels outdated.

While getting back into the routine of writing blog entries. I decided this time that I was going to use a Windows and MacOs simultaneously. Doing this to get used to both environments there might be a slight learning curve since I've spent most of my time building this website on a MacOs. One thing that I will have to figure out is the use of CodeKit. I used this on the MacOs to write my CSS code. Meaning that I wrote most of my styling in SASS therefore when put through Codekit it converted it into CSS. I picked up this form of writing code from DevTips way back in the day when Travis still hosted the show. So for now I will keep stylising to a minimum.

Will also have to point out things that happened when cloning the GithubPages Repo into my PC. When creating new files I've noticed that I have to "ADD" files to be tracked. It completely went over my head that I needed to do this. It's been such a long time that something simple as this stumped me.
